共享 NAS 存储使用方法分享

作者: 老蔡的一亩三分地 日期: 2020-05-27 阅读时间:3.0 分钟 字数:847

##前言
疫情防控期间，远程协同办公、外网会议需求明显上升，同时伴随着灾备领域新技术的不断发展、党建宣传工作力度的不断提升，工作中常常需要做外网资料搜集，为了方便大家外网非涉密资料的存储和分享，我在家中的NAS上部署了专用的共享分区，希望能对大家的工作有所帮助。
##更新日志
2020-05-22 V1.0 文档创建 by 蔡敏行
##快速上手
本节将介绍如何快速完成对 NAS 的一次性访问和文件操作。

在开始之前，推荐使用 Google Chrome 或 Firefox 浏览器访问 NAS， IE 浏览器不是一个好的选择，可能会因为性能原因导致访问或文件上传/下载卡顿，影响使用体验。
###接入 NAS
使用浏览器访问 https://quickconnect.cn/caichang01 或 https://caichang01.quickconnect.cn ，即可进入 NAS 的登陆界面。
[(img)]
输入账号：bybzb，密码：abc123，即可接入 NAS，进入控制台界面。
[(img)]
###浏览文件
在控制台界面，点击 File Station，进入文件浏览器界面。
[(img)]
文件浏览器中第一级目录共有三个文件夹， bybzb 账号对 home 和 homes 文件夹均为只读权限
###已有文件操作
在 File Station 中右键单击需要操作的文件，可以完成下载/重命名/压缩/电子文件附件共享等基于已有文件的操作。
[(img)]
###新增文件操作
在 File Station 上部工具栏中单击“上传”，可以选择本地文件进行上传操作。
[(img)]
上传操作分为 忽略 和 覆盖 两类，这是因为 NAS 的 DSM 系统并不支持断点续传操作，如果第一次上传文件中断，或目录中已有同名文件，选择“忽略”方式，原有文件将得到保留，本次上传的文件将以副本命名方式存在；选择“覆盖”方式，则原有文件将被覆盖。
对因上传失败而导致需要二次上传的文件，建议采取覆盖方式上传
###新建文件夹操作
在 File Station 上部工具栏中单击“新建文件夹”，即可完成新建文件夹操作。
##存储容量及建议
NAS 的共享存储容量为 100GB，出于提高利用效率，节约本地存储的考量，本文档建议存储的文件类型为：

  • 非密会议录音/录像等音视频文件
  • 常用工作软件安装包
  • 共享文档资料（如 .doc/ .ppt/ .xls/ .pdf 等格式）
    ##进阶操作
    部署共享存储的 NAS 使用的 DSM 系统拥有丰富的插件生态，可以实现一些非常有用的进阶操作。本节将介绍如何利用 DSM 系统的 Download Station 插件实现外网资料的远程下载，利用 Synology Office 插件实施在线文档协同编辑以及利用部署在本机的 Synology Drive Client 软件实现本地到 NAS 的文件双向同步，版本控制等进阶操作。
    ###外网资料远程下载
    在控制台界面，点击 Download Station，进入下载站控制界面 。
